Github

I dette prosjektet brukte vi GitHub for å lagre koden vår, spore endringer, og samarbeide effektivt som et team.
Hver av oss jobbet på våre egne brancher og brukte pull requests for å foreslå endringer.
Vi dokumenterte også fremdrift og oppgaver gjennom GitHub Issues.

Fordeler med GitHub

Liste opp noen av fordelene vi opplevde med GitHub:

  • Effektiv sporing av endringer med commit-historikk.
  • Muligheten til å samarbeide i sanntid uten å overskrive hverandres arbeid.
  • Bruk av Issues for å spore oppgaver og prioritere arbeidet.

Hvordan brukte vi Git med GitHub

For å jobbe med Git og GitHub fulgte vi disse enkle trinnene:

  1. Klon prosjektet til din lokale maskin: git clone <url-til-repositoriet>
  2. Opprett en ny branch for endringene dine: git checkout -b <branch-navn>
  3. Sjekk hvilke filer som er endret: git status
  4. Legg endringer til staging: git add .
  5. Lag en commit med en beskrivelse: git commit -m "Kort beskrivelse av endringen"
  6. Push endringene til GitHub: git push origin <branch-navn>
  7. Opprett en pull request fra GitHub for å slå sammen endringene med `main`.

Konflikter og Løsninger

I prosjektet møtte vi to hovedutfordringer som skapte konflikter:

  • Direkte commits til main: Enkelte glemte å oppdatere eller pull endringer fra `main` før de gjorde nye commits, noe som førte til konflikter ved merging.
    Løsning: Vi innførte en regel om alltid å oppdatere `main` først og teste endringer i en egen branch før de ble pushet.
  • Store bildefiler: Noen bilder hadde for stor filstørrelse, noe som skapte problemer ved commits og push.
    Løsning: Vi optimaliserte bildene med verktøy som TinyPNG før de ble lagt til i prosjektet.

Eksempler på Commit-Historikk

Commit-historikken vår viser hvordan vi jobbet systematisk med hyppige og små oppdateringer gjennom prosjektet.
Dette hjalp oss med å spore endringer og raskt identifisere feil når de oppsto.
